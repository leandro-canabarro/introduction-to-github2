## Passo 1: Criar um branch

_Bem-vindo(a)! :wave:_

**O que é GitHub?**: GitHub é uma plataforma de colaboração que usa _[Git](https://docs.github.com/get-started/quickstart/github-glossary#git)_ para versionamento. GitHub é um lugar popular para compartilhar e contribuir com software [open-source](https://docs.github.com/get-started/quickstart/github-glossary#open-source).
<br>:tv: [Vídeo: What is GitHub?](https://www.youtube.com/watch?v=pBy1zgt0XPc)

**O que é um repository?**: Um _[repository](https://docs.github.com/get-started/quickstart/github-glossary#repository)_ é um projeto contendo arquivos e pastas. Um repository rastreia versões de arquivos e pastas. Para mais informações, consulte "[About repositories](https://docs.github.com/en/repositories/creating-and-managing-repositories/about-repositories)" no GitHub Docs.

**O que é um branch?**: Um _[branch](https://docs.github.com/en/get-started/quickstart/github-glossary#branch)_ é uma versão paralela do seu repository. Por padrão, seu repository tem um branch chamado `main` e ele é considerado o branch definitivo. Criar branches adicionais permite que você copie o branch `main` do seu repository e faça alterações com segurança sem interromper o projeto principal. Muitas pessoas usam branches para trabalhar em recursos específicos sem afetar outras partes do projeto.

Branches permitem que você separe seu trabalho do branch `main`. Em outras palavras, o trabalho de todos está seguro enquanto você contribui. Para mais informações, consulte "[About branches](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-branches)".

**O que é um profile README?**: Um _[profile README](https://docs.github.com/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme)_ é essencialmente uma seção "Sobre mim" no seu perfil do GitHub onde você pode compartilhar informações sobre você com a comunidade no GitHub.com. O GitHub mostra seu profile README no topo da sua página de perfil. Para mais informações, consulte "[Managing your profile README](https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme)".

### :keyboard: Atividade: Seu primeiro branch

1. Abra uma nova aba do navegador e navegue até seu repository recém-criado. Em seguida, trabalhe nas etapas em sua segunda aba enquanto lê as instruções nesta aba.
2. Navegue até a aba **< > Code** no menu de cabeçalho do seu repository.

   ![code-tab](/images/code-tab.png)

3. Clique no menu suspenso do branch **main**.

   ![main-branch-dropdown](/images/main-branch-dropdown.png)

4. No campo, nomeie seu branch como `my-first-branch`. Neste caso, o nome deve ser `my-first-branch` para acionar o workflow do curso.
5. Clique em **Create branch: my-first-branch** para criar seu branch.

   ![create-branch-button](/images/create-branch-button.png)

   O branch mudará automaticamente para o que você acabou de criar.
   A barra suspensa do branch **main** refletirá seu novo branch e exibirá o novo nome do branch.

6. Aguarde cerca de 20 segundos e atualize esta página (aquela de onde você está seguindo as instruções). O [GitHub Actions](https://docs.github.com/en/actions) atualizará automaticamente para o próximo passo.
